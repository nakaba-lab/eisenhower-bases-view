# Changelog

このプロジェクトの注目すべき変更を記録する。書式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に従い、バージョニングは [Semantic Versioning](https://semver.org/lang/ja/) に従う。

## [Unreleased]

### Added

- Bases カスタムビュー「Eisenhower Matrix」の登録（アダプタ層）。Bases が無効な環境では graceful に処理してプラグインの他機能を壊さない。ビューはローディング・空・初期状態のシェルを表示する（4 象限の配置は今後のリリースで追加）。(#18)
- 各ノートの緊急度・重要度（boolean 軸）を読んで 2×2 マトリクス（Do/Schedule/Delegate/Delete）＋未分類ゾーンへ自動配置。軸プロパティが未設定（absent）のノートは `false` と区別して未分類ゾーンに表示する。テーマ追従・レスポンシブ・各象限の空状態表示に対応。(#19)
- カードをドラッグして象限を変更すると、両軸を frontmatter に書き戻して分類を永続化する（マウス・キーボード操作対応）。移動はすぐ画面に反映し（楽観的更新）、書き込みに失敗したら元に戻して通知する。未分類ゾーンはドロップ先にならず、未分類カードを象限へドラッグすると分類できる。(#20)
- ビューの Configure view で緊急度軸・重要度軸に使う frontmatter プロパティを選べるようにした（ビュー設定を主、プラグイン設定を未設定時のデフォルトにするハイブリッド）。選択候補は書き戻し可能な `note.*` プロパティのみに絞り（`formula`/`file` は除外）、軸を変更すると新しいプロパティに基づいて再配置される。書き戻せない軸が指定された状態でのドラッグは通知して弾き、frontmatter を壊さない。(#21)
- カードからノートを開けるようにした: クリック（または Enter）で現在のタブ、Cmd（mac）/Ctrl（win）+クリック（または Enter）で新しいタブに開く。ホバーで Obsidian コアのページプレビューを表示する（実際に出すかはコアの「ページプレビュー」設定に従う）。キーボードは Tab でカードにフォーカス（フォーカスリング表示）し、Enter で開く・Space で掴んで移動、と割り当てを整理した。(#22)
- プラグイン設定タブを追加: デフォルトの緊急度/重要度プロパティ、象限ごとのラベルと色、欠損ノートの未分類表示、表示言語（Auto/英語/日本語）を設定できる。変更は開いているマトリクスへ即時反映され、再起動後も保持される。表示言語は Auto で Obsidian の言語に追従し、マトリクスの UI 文言・既定の象限ラベルに加え、設定タブ・Bases の軸プロパティ選択 UI・通知メッセージ（Notice）も英/日で切り替わる。象限の色は未設定ならテーマのアクセント色に追従する。(#23)
- 直前の移動を「元に戻す」機能を追加: カードをドラッグして分類し直した直後に、ビュー内の「元に戻す」トースト、またはコマンド「Eisenhower Matrix: 直前の移動を元に戻す」（任意のホットキーを割り当て可能）で 1 手戻せる。元が未分類だったカードは緊急度・重要度プロパティを削除して未分類へ戻す（完全復元）。Obsidian 標準の取り消し（Ctrl+Z）とは独立した専用の取り消しで、保持するのは直前の 1 手のみ（多段・やり直しはなし）。

### Fixed

- 軸プロパティが未設定（absent）のノートが、未分類ゾーンではなく Delete 象限に誤って配置される不具合を修正（未リリースの #19 機能の不具合）。実機 Obsidian の欠損値（`NullValue`）判定を文字列表現依存から型同一性（`instanceof`）へ是正し、実機 E2E で未分類化を確認した。(#33)
- 軸に非 boolean のプロパティ（数値・文字列の `note.*`）を指定したノートが 4 象限に分類されてドラッグ可能になり、ドロップで両軸が `true`/`false` に上書きされて元の値を破壊しうる不具合を修正（未リリースの #19 機能の不具合）。v1 は boolean 軸限定のため、値が boolean のプロパティだけを分類し、非 boolean（および absent）のノートは未分類ゾーンへ退避するようにした。(#34)
- Base 自身の `.base` ファイルや `.canvas`・画像などの非 Markdown が「未分類」カードとして現れうる問題を修正（未リリースの #19 機能の挙動改善）。配置対象を Markdown ノート（`file.extension === "md"`）のみに限定し、非 Markdown は象限にも未分類にも出さないようにした。Markdown ノートが 1 件も無い Base は空状態を表示する。
- 軸に数値・文字列など boolean 以外の値を持つノートを未分類ゾーンから象限へドラッグすると、両軸が `true`/`false` で上書きされ元の値が壊れる問題を防いだ（未リリースの #19/#34 機能の穴を封鎖）。そうしたノートは未分類ゾーンで淡色＋鍵アイコン付きの**ドラッグ不可カード**として表示し、誤ドロップによるデータ破壊を防ぐ（クリックで開いて値を直せる）。軸プロパティが未設定（欠損）のノートは従来どおりドラッグして分類できる。
- 表示まわりの細かな改善（いずれも未リリース機能の詰め）: Bases が無効な環境での登録失敗メッセージも表示言語（英/日）に追従するようにした。象限の件数バッジをスクリーンリーダーが読み上げられるようにした。英語表示時の件数を単複で正しく出す（`1 item` / `5 items`）。
- さらに未リリース機能の詰め: カードを移動した直後に「元に戻す」トーストで取り消すと、カードが移動先の象限に貼り付いて見える場合があったのを修正（ファイル自体は正しく戻っていた）。緊急度・重要度の両軸に同じプロパティを割り当てるとドラッグで値が壊れうるため、その設定では移動を通知して弾くようにした。未分類ゾーンを非表示にしていて表示できるカードが全て未分類のとき、画面が無言で空になる代わりに件数ヒントを出すようにした。設定の言語「Auto」を日本語 UI では「自動」と表示。ビューを狭いペイン（サイドバー等）に置いたときも 2×2 が 1 列に畳まれるよう、ペイン幅に追従するようにした。「元に戻す」トーストのボタンをキーボードで操作した後、フォーカスが失われずマトリクスへ戻るようにした。
- カードをドラッグ中、カーソルに追従する浮遊カード（オーバーレイ）が実際のカーソル位置から一定量ずれて表示される問題を修正（未リリースの #20 機能の不具合）。ビューを中央ペイン・サイドパネルのいずれに置いても、掴んだ位置にオーバーレイが正しく追従するようにした。(#43)
- ビューをポップアウトウィンドウ（別ウィンドウ）に出すとカードを掴めずドラッグを開始できない問題を修正（未リリースの #20 機能の不具合）。別ウィンドウでもカードを掴んで象限を移動できるようにした（マウス・キーボード操作とも）。実機 Obsidian で修正前後を確認（別ウィンドウでの掴み: 不可→可、メインウィンドウは変化なし）。(#44)
