# 環境チェック（軽量）

> 対象フロー: 既存のみ ｜ 種別: auto

既存プロジェクトは「環境バリデーション」タスク（新規のみ）を通らないため、後続が前提とするツールの存在をここで軽量に確認する（無いと Claude Code フックの実行・docs サイトの検証・リモート操作が欠けたまま無言で失敗・劣化する）。各項目の確認コマンド・導入案内は「環境バリデーション」タスクの表を参照し、ここで重複定義しない。対象は次の 4 項目に絞る：

| 項目 | 確認する条件 |
|------|-------------|
| Python ランタイム（python3） | 常に（Claude Code フック〔`.claude/hooks/*.py`〕の実行に必須） |
| git | 常に（バージョン管理・`.githooks` の git フックの前提） |
| Node.js / npm | docsSite を使う場合のみ（`.claude/project-profile.json` の `docsSite.enabled`＝出荷時 true。要否の最終確定は「docs サイト有効化」タスク） |
| VCS CLI（gh / glab） | リモートがある場合のみ（`git remote -v` で判定。使うホスト側だけでよい） |

- 結果を表（OK / 未設定）で提示する。条件に該当しない項目（docs サイトを使わない・リモートが無い）は「対象外」と明示してよい。
- **未充足あり** → 導入を促し、導入しないまま進む場合は利用者の明示承認（「このまま進む」）を得てから先へ進む（「環境バリデーション」タスクと同じ作法）。

`【完了条件】対象 4 項目の確認表（OK / 未設定。条件に該当しない項目は「対象外」と明示）を提示し、全項目が OK・対象外、または未充足項目に対し利用者が「このまま進む」と明示承認した。`
