# 用語集

業務用語（日本語）と、コード上で使う英語識別子の対訳。**同一概念に複数の英語名（訳語のブレ）を生まない**ための正。新しい業務概念を命名したらこの表に追記してから使う（ローマ字識別子は禁止＝`post_edit.py` が検知。英語名の正はここに置く）。

| 用語（日本語） | 英語名（識別子） | 使用箇所・備考 |
|----------------|------------------|----------------|
| 象限 | quadrant | 緊急×重要の 4 区分（Do/Schedule/Delegate/Delete）。`classifyQuadrant` |
| 緊急度（軸） | urgency (axis) | 横軸。boolean プロパティ（v1） |
| 重要度（軸） | importance (axis) | 縦軸。boolean プロパティ（v1） |
| 軸プロパティ | axis property | 象限算出に使う `note.*` の frontmatter プロパティ |
| 未分類ゾーン | unclassified zone | 軸欠損（absent）ノートの置き場。ドロップ不可 |
| 欠損 / 未定義 | absent | プロパティ未定義。`false` とは区別する |
| 書き戻し | write-back | ドラッグで frontmatter を更新する操作。`processFrontMatter` |
| 楽観的更新 | optimistic update | 書き込み確定前に UI を先行更新し、失敗時ロールバック |
| エントリ | entry | Bases が返す 1 ノート分のデータ（`BasesEntry`） |
| ビュー設定 | view options / settings | Bases ビュー options（主）＋プラグイン設定（デフォルト） |
| 象限ラベル | quadrant label | 各象限の見出し。カスタム空は言語既定にフォールバック（F6）。`quadrantLabels` |
| 象限色（アクセント色） | quadrant color / accent color | 象限の左ボーダー色。空はテーマ既定（`--interactive-accent`）追従（F6）。`quadrantColors` |
| 表示言語 | language | UI 文言・既定象限ラベルの言語。`auto`（Obsidian 追従）/`en`/`ja`（F6）。`resolveLanguage` |
| 表示情報 | presentation | UI へ渡す解決済みラベル/色/言語文言の束（F6）。`MatrixPresentation` |
| バッジ | badge | カードにタイトル下へ表示する読み取り専用の追加プロパティ（期日・タグ等）（F8）。`MatrixEntry.badges`／`readBadges` |
| カード表示プロパティ | card badge property | バッジに表示する追加プロパティ。読み取り専用のため `formula.*`／`file.*` も可（F8）。`cardBadgeProperties`／`resolveBadgePropertyIds` |
| 日付強調 | date emphasis | 厳格 ISO（`YYYY-MM-DD`）で今日以前のバッジをアクセント強調する任意機能（F8・既定オフ）。`emphasizePastDates`／`isEmphasizedDate` |
| 診断（情報） | diagnostics | 設定ミス（両軸同一キー）・解決済み軸名など、UI へ提示する診断情報（F7）。既存の軸解決結果を転送する。`MatrixDiagnostics` |
| 滞留 | stagnation / stagnant | 最終更新（`mtime`）から一定日数を超えて動いていない状態。バッジで可視化（F9）。`evaluateStagnation`・`stagnant`・`stagnantDays` |
| 滞留しきい値 | stagnation threshold | 滞留とみなす日数（既定 14・0 でオフ）。ビュー options 主＋設定既定のハイブリッド（F9）。`stagnationThresholdDays` |
| 完了トグル | completion toggle | カードのチェックで完了（`done: true`）を書き戻す機能（F10）。双方向・`delete` しない。`writeCompletion`／`onToggleCompletion` |
| 完了プロパティ | completion property | 完了を書き込む boolean の `note.*`（**既定 `done`＝初期状態で有効**・空で無効化）。軸と同一キーは弾く（F10）。`completionProperty`／`resolveCompletionId` |
| 完了ノートの淡色表示 | dim completed | `done != true` フィルタを張らない利用者向けに完了カードを弱色トークンで淡色化する表示オプション（F10・既定オフ）。`dimCompleted` |
| 軸仕様 | axis spec | 軸の解釈と書き戻し方法を規定する型（boolean/number/select/tag）（v0.3・#120）。`AxisSpec` |
| 軸の生値 | axis raw | アダプタが Obsidian `Value` を型正規化した生値（型タグ付き判別共用体・absent/boolean/number/string/array）（#120）。`AxisRaw` |
| 軸の側 | axis side | 軸値を二分した結果（`true`/`false`／未分類 `undefined`）。象限算出の安定インターフェース（#120）。`interpretAxis` の `side` |
| （軸の）ロック | locked | present だが軸仕様で解釈できない値。ドラッグ不可にして既存値を保護（#34 を軸タイプ一般へ拡張・#120）。数値軸は 1a では常時 locked だったが #122 v0.3-1b で有限数を unlock（掴める＝書き戻し解禁）＝現在 locked は非有限・型不一致・off-sentinel（しきい値未設定）・未対応値のみ。`AxisReading.locked` |
| 数値しきい値（軸） | number threshold (axis) | 数値軸を二分するしきい値（`value >= threshold` で高い側＝境界ちょうども高い側）。ビュー options 主＋設定既定のハイブリッド・per-axis（空＝未設定＝数値軸オフ）（v0.3-1a・#121）。`NumberThresholds`／`resolveNumberThresholds`／`defaultUrgencyThreshold` |
| 選択値（軸） | select values (axis) | 選択（文字列）軸の 2 値（`trueValue`＝真側／`falseValue`＝偽側）。完全一致で象限側・どちらとも不一致の未知値（3 値目 `medium` 等）は locked（**二値割り切り＝採択 A**）。ビュー options 主＋設定既定のハイブリッド・per-axis（**両値が非空かつ互いに異なるとき有効**・同値/片方空はオフ）（v0.3-2・#123）。`SelectValues`／`resolveSelectValues`／`defaultUrgencySelectTrueValue` |
