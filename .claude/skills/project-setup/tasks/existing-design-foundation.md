# デザインの土台の導入（Web のみ）

> 対象フロー: 既存のみ ｜ 種別: 【人間ゲート】

> 新規フローの「雛形構築」タスクが立てる「デザインの土台」(a)〜(d) の**既存プロジェクト版**。既存フローは環境バリデーションも雛形構築も通らないため、これが無いと既存 Web プロジェクトはデザイントークン・コンポーネントカタログ・a11y lint・基線スクリーンショットを一切持たないまま開発フロー（`/dev-tasks`）に入り、Claude が裸でフロントを書くことになる。「プロファイル確定」タスクの**後**に置く（kind・frontendDir が確定済みのため対象判定ができる）。

1. **対象判定**: profile の `kind` が web 以外、または `frontendDir` が `"none"`（UI を持たない明示）なら「対象外」＝profile の値を証拠として提示し completed にする（no-op）。
2. **(0) ブラウザ環境確認**: Google Chrome 本体（または `playwright` 同梱の Chromium）の存在を確認する。無ければ導入を案内し、**確認できないまま (d) に進まない**（`chrome-devtools` MCP は Chrome 本体を要する。新規フローの環境バリデーションに相当する確認をここで行う）。
3. **(a) デザイントークン**: 既存コードベースにトークン定義（テーマファイル・デザインシステム・CSS 変数等）が**既にあるかを先に探し**、あれば流用を記録する。無ければ最小のトークン定義ファイルの新設を提案する（JSON 等に集約すると設計書のトークン可視化が使える＝`.claude/rules/design-doc.md`）。
4. **(b) コンポーネントカタログ**: 既存に Storybook 等があるか確認する。無ければ `AskUserQuestion`（options:「導入する（推奨・既定 Storybook）／同等のプレビュー手段に読み替える／opt-out（理由を設計書の UI 領域に記録）」）で採否を確認し、**承認時のみ**導入する。
5. **(c) a11y lint**: フロントのスタックに応じて `/setup-js`（React/JSX なら `eslint-plugin-jsx-a11y`）等で導入する。profile の `checks` への追記は**「プロファイル確定」タスクで確定済みの値に追補**する形で行う（`.claude/` 配下＝auto mode オフ）。**既存プロジェクトは既存 ESLint・フレームワーク管理 lint（`eslint-config-next` 等）を持つことが多い**ため、`/setup-js` は既存設定を検出したら無条件に被せず `AskUserQuestion` で方針を確認する（同スキルの「既存設定・フレームワークの検出」ガード＝非破壊）。
6. **(d) 基線スクリーンショット**: `commands.dev` を Bash の `run_in_background` で起動 → `curl` で readiness 確認 → `playwright` で主要画面を**モバイル幅とデスクトップ幅**で撮り、`docs/screenshots/baseline-<画面>-<ブレークポイント>.png` に保存する（以後の `frontend-reviewer` の比較基線。`commands.dev` 未設定なら「プロファイル確定」タスクに戻って確定してから行う）。
7. **(e) コンポーネントテスト環境【必須】**: コンポーネントの描画/操作テストが走る環境（DOM 環境。JS なら jsdom 等）が既存にあるか確認し、無ければ**最小の描画テスト 1 本が緑**になる環境を導入する（(b) カタログと同じ「既定〔DOM テスト環境＋Testing Library 等〕＋同等手段に読み替え＋成立しないなら opt-out 理由を設計書の UI 領域に記録」）。`.claude/rules/tdd.md` が要求する「画面要素 AC のレンダリング/操作テスト」を土台から書けるようにする（DOM 環境が無いまま描画テストを書いて落ちる事故を防ぐ）。`checks`/設定は `.claude/` 配下＝auto mode オフで「プロファイル確定」の値に追補する。

`【完了条件】対象外なら profile の根拠（kind／frontendDir）を提示。対象なら ブラウザ確認結果＋(a) トークン定義のパス（既存流用 or 新設）＋(b) カタログ起動ログ（または opt-out の記録先）＋(c) lint 実行ログ＋(d) 基線スクショのファイルパス＋(e) 最小描画テストが緑の実行ログ（または opt-out の記録先）、を提示。【証拠なしに completed 禁止】承認・証拠の前に completed にしない。`
