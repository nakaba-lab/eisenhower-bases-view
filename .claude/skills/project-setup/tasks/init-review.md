# `/init` 実行＋追記確認

> 対象フロー: 新規・既存とも ｜ 種別: 【人間ゲート】

> 手順は CLAUDE.md への追記のみ（`.claude/` 配下は書かない）ため、ここでは auto mode の制約はかからない。

1. `/init` を実行し、コードベース情報を CLAUDE.md に追記する。既存プロジェクトの場合、コードベースから判明したビルド/テスト/Lint コマンド・言語・フレームワークも控える（プロファイル反映は「プロファイル確定」タスク）。
2. **追記レビューゲート（`AskUserQuestion`）**：次の recommender はこの追記を前提に自動化を提案するため、誤検出（言語取り違え・古いコマンド）のまま渡さないよう確認する。
   - header: `/init 追記`
   - question:「`/init` が CLAUDE.md にコードベース情報（言語・構成・主要コマンド）を追記しました。次の recommender はこの内容を前提に提案します。追記内容は妥当ですか？」
   - options（最大 4・**multiSelect は使わない**＝選別リストが無く可否＋軽い訂正だけ）:「このまま進む（推奨）／検出を誤りを直す（言語・コマンド等を指摘 → Claude が該当箇所だけ修正）／追記を補う（不足情報を足す）」

`【完了条件】/init 完了で CLAUDE.md に追記された差分の要点を提示し、AskUserQuestion の回答（妥当 or 修正指示）を得た。【証拠なしに completed 禁止】回答を得る前に completed にしない。`
