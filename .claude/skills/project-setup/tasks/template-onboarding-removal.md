# テンプレ導入ブロックの削除

> 対象フロー: 新規・既存とも ｜ 種別: auto

> CLAUDE.md を更新する生成系（`/init`・recommender・improver）が走る**前に**、出荷時の自己言及ブロックを除去しておく。このブロックは生成系の追記アンカー（CLAUDE.md 冒頭「コードベース固有情報は `/init` で追記」の直下にある最初の `##` 節）の位置にあり、残したままだと生成系がブロック内へ書き込み、後段の検証で巻き添え削除される恐れがある。**`/init` の前**に消すことでこの罠を断つ。CLAUDE.md はルート直下（`.claude/` 外）のため auto mode 制約は受けない。

出荷時の CLAUDE.md 冒頭には、テンプレート初回利用者への導入（`TEMPLATE-ONBOARDING:START` を含む行〜`TEMPLATE-ONBOARDING:END` を含む行で囲まれた「## このテンプレートについて」節）が入っている。これは「自分はテンプレートである」という自己言及メタで、適合済みプロジェクトには不要かつ事実と食い違う（例:「まず最初に `/project-setup` を実行」はセットアップ後には古い）。

- **`TEMPLATE-ONBOARDING:START` を含む行から `TEMPLATE-ONBOARDING:END` を含む行まで（両端のマーカ行を含む）を丸ごと削除する**（START 行には説明文が付くので、完全一致ではなく `TEMPLATE-ONBOARDING:START`／`:END` のトークンで照合する）。削除後は **イントロ文と「## プロジェクト設定」節の間が「空行 → `---` → 空行」1 組だけ**になるよう、余分な空行を 1 行にまとめる。

`【完了条件】削除差分を提示し、CLAUDE.md に TEMPLATE-ONBOARDING マーカが 1 つも残っていない（grep 相当で確認）こと、イントロ文と「## プロジェクト設定」節の間が「空行 → --- → 空行」1 組であることを提示。`
