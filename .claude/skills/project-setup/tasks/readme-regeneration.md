# README をプロジェクト向けに再生成

> 対象フロー: 新規・既存とも ｜ 種別: 【人間ゲート】

> README はルート直下（`.claude/` 外）のため auto mode 制約は受けない。テンプレ自己言及の除去（「テンプレ導入ブロックの削除」「テンプレ自己言及の最終検証」の両タスク）は CLAUDE.md だけが対象で、README には触れていない。本タスクが README を担当する。

出荷時 README はテンプレート自身の説明（タイトル「Claude Code エージェント開発ワークフロー・テンプレート」、「Release の zip として配布」、「固定の実例を同梱しない」等）で、適合済みプロジェクトには不要かつ事実と食い違う。これをプロジェクト固有の README に再生成する。

1. **対象判定**：
   - **新規プロジェクト**：常に全文を再生成する（判定不要）。
   - **既存プロジェクト**：README 冒頭の `<!-- TEMPLATE-README` センチネル（無ければフォールバックでタイトル「…ワークフロー・テンプレート」等のテンプレ固有文字列）が**あるときだけ**再生成する。無ければ＝利用者独自 README とみなして**触らず**、「利用者の README を保持した」と報告して完了する（誤って利用者の README を書き換えない）。
2. **材料を集める**（言語・FW 非依存。固定のスタック文面は持たない）：
   - **プロジェクト名**：新規＝要件定義書／ヒアリング、既存＝ディレクトリ名 or CLAUDE.md。
   - **説明**：新規＝要件定義書の「背景・目的」、既存＝`/init` が CLAUDE.md に追記したコードベース要約。
   - **コマンド・パッケージマネージャ・VCS**：`.claude/project-profile.json`（「プロファイル確定」タスクで確定済み）。
3. **再生成する**（空欄のコマンドは行ごと省く）：
   - `# <プロジェクト名>` ＋ 何をする/何を解決するかの 1〜2 文。
   - `## セットアップ`：`packageManager` での依存インストール。
   - `## 開発コマンド`：profile の `commands.build` / `test` / `lint` / `format` を表で（空の項目は行ごと省く）。
   - `## 開発ワークフロー`：「Issue 駆動 + Git Flow + worktree + TDD + Spec 駆動 + 多段レビューを採用。詳細は `CLAUDE.md`・`.claude/rules/` を参照」という**短いポインタ**にとどめる（テンプレ説明の丸写しはしない＝ワークフロー解説の本体は CLAUDE.md 側に残す）。
   - `## 前提`：Claude Code CLI ／ フックに python3 ／ VCS は `vcsHost` に応じて gh または glab。
   - **末尾に由来行**：`.claude/project-profile.json` の `templateVersion`・`templateOrigin` を読み、「このプロジェクトは Claude Code エージェント開発ワークフロー・テンプレート v`<templateVersion>` から生成」の 1 行を README 末尾に入れる（`templateOrigin` が非空なら「不具合・改善要望は `<templateOrigin>` へ」を続ける。**両方が空ならこの行ごと省略する**）。テンプレ更新の取り込み・不具合報告の導線になる出自の記録であり、手順 5 の自己言及検査の対象外。
4. **人間ゲート（`AskUserQuestion`）**：
   - header: `README 再生成`
   - question:「README をプロジェクト向けに再生成しました。この内容で確定してよいですか？」
   - options（**multiSelect は使わない**）:「確定する（推奨）／修正する（指摘箇所を直して再提示）」
5. 承認後に書き込みを確定し、`TEMPLATE-README` センチネルとテンプレ自己言及文（テンプレタイトル・「Release の zip として配布」・「固定の実例を同梱しない」等）が 1 つも残っていないこと（grep 相当）を確認する（手順 3 で入れた**末尾の由来行は出自の記録としてこの検査の対象外**）。

`【完了条件】再生成後 README に旧テンプレ自己言及（TEMPLATE-README センチネル／テンプレタイトル／「Release の zip として配布」「固定の実例を同梱しない」等）が末尾の由来行を除いて 1 つも残っておらず、プロジェクト名・説明・profile 由来コマンド（および由来行＝templateVersion/templateOrigin が空なら省略）が反映され、利用者が AskUserQuestion で承認したことを提示。既存で独自 README の場合は「触らない」と判断した旨を提示。【証拠なしに completed 禁止】承認回答を得る前・書き込みを確定する前に completed にしない。`
