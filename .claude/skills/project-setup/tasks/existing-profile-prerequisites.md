# プロファイル前提値の確定（VCSホスト・フロントエンド構成）

> 対象フロー: 既存のみ ｜ 種別: auto

既存プロジェクトはヒアリング充足ループも雛形構築も（いずれも新規のみ）通らないため、`vcsHost` と `frontendDir` が未確定。プロファイルに書くため、profile 確定の前段でここを確定する（どちらも「profile 前段の確定値」という単一役割。**片方だけ確定して completed にしない**）：

1. **vcsHost**: `git remote -v` のホストから判定し、remote が無ければユーザーに確認する。
2. **フロントエンド構成（`frontendDir`）**: このタスク時点では profile の `kind` が未確定のため、**コードベースから自力で Web 相当かを判定**する（根拠＝「/init 実行＋追記確認」タスクで控えたフレームワーク、UI ファイル〔コンポーネント・テンプレート・スタイルシート〕の存在）。Web 相当なら UI コードのあるディレクトリを特定する（**リポジトリルート直下にフロントがあるなら `"."`**）。UI を持たない（API 専用等）なら **`"none"`（明示 opt-out）を提案**する。空（未確認）のまま「プロファイル確定」タスクへ流さない。

`【完了条件】vcsHost（github/gitlab）の確定根拠と、frontendDir の確定根拠（UI ファイルの所在。または "none" と判断した根拠）の両方を提示。`
