## [0.2.0] - 2026-07-11

### Added

- カード上の**完了トグル**を追加した（#105）。設定した完了プロパティ（boolean・例 `done`・既定は空＝オフの opt-in）が有効なとき、カードにチェックボタンが出て、クリック（またはカードにフォーカス中に `x` キー）で「完了（`done: true`）」を書き込める。終わったタスクをノートを開かずにカード上で片付けられ、Do 象限を「今やること」だけに保てる。完了ノートを消すか残すかは Base 側のフィルタに委ねる方針で、Base に `done != true` を張れば完了したカードは自動で消える（README に推奨フィルタを記載予定）。フィルタを張らない場合の目印として「完了ノートを淡色表示」オプションも用意した。チェックは押すたびに完了/未完了を切り替えられ、直前の 1 手はドラッグ移動と同じ「元に戻す」で戻せる。完了プロパティが日付など boolean 以外の値を持つノートでは、既存の値を壊さないようチェックを無効化する。**既定はオフ**なので、これまでのカードの見た目・挙動は変わらない。
- 滞留インジケータ（#106）: 最終更新から一定日数（既定 14 日・設定タブで変更、0 で無効）を超えて動いていないカードに、控えめな滞留バッジ（時計アイコン＋経過日数・淡色）を付けて「見直すべき古いカード」を一目で分かるようにした。ノートは一切書き換えない読み取り専用の目印で、判定は最終更新日時（mtime）に基づくため、Linter・Templater・同期などの自動処理でも更新日時はリセットされる点に注意（設定画面に明記）。
- カードにタイトルの下へ、期日・タグ・プロジェクトなどの追加プロパティを**読み取り専用のバッジ**で表示できるようにした（最大 3 個）。ホバープレビューを開かなくても期日などが一目で分かり、Do↔Schedule の振り分けがカード上で完結する。表示するプロパティはビューごと（＋設定でデフォルト）に選べ、読み取り専用なので数式や `file.*` 系プロパティも表示できる。期日らしい値（`YYYY-MM-DD` で今日以前）を強調するトグルも用意した（既定オフ）。**既定は表示 0 個**なので、これまでのカードの見た目は変わらない（#104）。
- ビュー内に**設定ミス診断バナー**と**解決済み軸名の表示**を追加した（#103）。緊急度軸と重要度軸に同じプロパティを割り当ててしまい全カードが移動不可（🔒）になる設定ミスを、原因（同一プロパティ名）と直し方（2 軸に別々のプロパティを指定）付きの警告バナーで画面に説明する。あわせて、ノートが 0 件のときや軸プロパティ名の打ち間違いで全ノートが未分類に落ちたときに、いまどの緊急度／重要度プロパティで分類しているかを控えめに 1 行表示する。配色は Obsidian テーマに追従し（ライト/ダーク両対応）、既存の軸解決結果を再利用するだけでプラグインの動作・書き込みは変えない。

### Changed

- 依存パッケージを更新した（プラグインの挙動は不変）。ランタイム依存 preact を 10.29.3 → 10.29.4（patch）に更新した（preact は配布バンドルに取り込まれるため次回ビルドの `main.js` は preact 10.29.4 を含むが、機能上の変化はない）。開発ツール（vitest 2 → 4・jsdom 25 → 29・@types/node 22 → 26・typescript-eslint 8.62.1）も更新した（`main.js` には無影響）。develop に載せた結合状態で typecheck・273 テスト・lint・production build がすべて緑であることを確認済み（Dependabot #49, #50, #53, #54, #55 を統合）。

## [0.1.7] - 2026-07-06

### Fixed

- 多角的レビュー（多観点＋敵対的検証）で確定した指摘に対応した（挙動の堅牢化・a11y 改善。プラグインの主要な使い勝手は不変）。
  - スクリーンリーダー/キーボード利用者に、移動成功の読み上げへ「元に戻す」導線（コマンド「直前の移動を元に戻す」）への案内を添え、非表示・自動消滅するビュー内トーストに気づけなくても undo に到達できるようにした（発見可能性の視覚/非視覚パリティ）。
  - Bases API 接触点（軸読み取り `getValue`・軸解決 `getAsPropertyId`）が万一例外を投げても、マトリクス全体の描画が巻き添えで壊れないようにした（`getValue` の例外はその軸だけ未分類へ退避し、**書き戻し可能軸で型を確証できない場合は安全側でドラッグ不可**にして非 boolean 値の上書き破壊を防ぐ。`getAsPropertyId` の例外は既定軸へフォールバック。Bases API 変更への耐性）。
  - ドラッグ中にビュー（リーフ）を閉じた場合に、書き戻し完了後の後処理がアンマウント済みのビューで走り、掃除されないタイマーが残る問題を防いだ。
  - 書き戻し成功の確定タイミングと再描画が競合したとき、楽観的に移動したカードのオーバーレイが残って以後のサーバ値変更を隠すことがある問題を、成功確定時の再突合で解消した。
  - コマンドパレット経由の「元に戻す」で、ファイルは戻ったのにカードが移動先象限へ貼り付いて見えることがある残存を解消した（ビュー内トースト経由の undo と挙動を対称化）。
- テストの堅牢化: ドラッグ書き戻しの成否確定（settle）の分岐を純関数 `planSettle` に切り出して結線を単体固定、ポップアウトのキーボードドラッグ経路（空リスナー配列の張り替え）・aria-live 差分化の初回契約・軸読み取りの例外退避を追加でテスト、フレーキーだった 500 件の実時間アサーションを分類正しさの固定へ置き換えた。

### Changed

- 開発ツールを追加した（プラグインの挙動・配布バンドル `main.js` は不変）。`eslint-plugin-obsidianmd` を導入し、Obsidian コミュニティプラグイン申請時の bot 検証相当を提出前に lint で先取りできるようにした（#80）。

## [0.1.0] - 2026-07-05

### Added

- Bases カスタムビュー「Eisenhower Matrix」の登録（アダプタ層）。Bases が無効な環境では graceful に処理してプラグインの他機能を壊さない。ビューはローディング・空・初期状態のシェルを表示する（4 象限の配置は今後のリリースで追加）。(#18)
- 各ノートの緊急度・重要度（boolean 軸）を読んで 2×2 マトリクス（Do/Schedule/Delegate/Delete）＋未分類ゾーンへ自動配置。軸プロパティが未設定（absent）のノートは `false` と区別して未分類ゾーンに表示する。テーマ追従・レスポンシブ・各象限の空状態表示に対応。(#19)
- カードをドラッグして象限を変更すると、両軸を frontmatter に書き戻して分類を永続化する（マウス・キーボード操作対応）。移動はすぐ画面に反映し（楽観的更新）、書き込みに失敗したら元に戻して通知する。未分類ゾーンはドロップ先にならず、未分類カードを象限へドラッグすると分類できる。(#20)
- ビューの Configure view で緊急度軸・重要度軸に使う frontmatter プロパティを選べるようにした（ビュー設定を主、プラグイン設定を未設定時のデフォルトにするハイブリッド）。選択候補は書き戻し可能な `note.*` プロパティのみに絞り（`formula`/`file` は除外）、軸を変更すると新しいプロパティに基づいて再配置される。書き戻せない軸が指定された状態でのドラッグは通知して弾き、frontmatter を壊さない。(#21)
- カードからノートを開けるようにした: クリック（または Enter）で現在のタブ、Cmd（mac）/Ctrl（win）+クリック（または Enter）で新しいタブに開く。ホバーで Obsidian コアのページプレビューを表示する（実際に出すかはコアの「ページプレビュー」設定に従う）。キーボードは Tab でカードにフォーカス（フォーカスリング表示）し、Enter で開く・Space で掴んで移動、と割り当てを整理した。(#22)
- プラグイン設定タブを追加: デフォルトの緊急度/重要度プロパティ、象限ごとのラベルと色、欠損ノートの未分類表示、表示言語（Auto/英語/日本語）を設定できる。変更は開いているマトリクスへ即時反映され、再起動後も保持される。表示言語は Auto で Obsidian の言語に追従し、マトリクスの UI 文言・既定の象限ラベルに加え、設定タブ・Bases の軸プロパティ選択 UI・通知メッセージ（Notice）も英/日で切り替わる。象限の色は未設定ならテーマのアクセント色に追従する。(#23)
- 直前の移動を「元に戻す」機能を追加: カードをドラッグして分類し直した直後に、ビュー内の「元に戻す」トースト、またはコマンド「Eisenhower Matrix: 直前の移動を元に戻す」（任意のホットキーを割り当て可能）で 1 手戻せる。元が未分類だったカードは緊急度・重要度プロパティを削除して未分類へ戻す（完全復元）。Obsidian 標準の取り消し（Ctrl+Z）とは独立した専用の取り消しで、保持するのは直前の 1 手のみ（多段・やり直しはなし）。

### Fixed

- 軸プロパティが未設定（absent）のノートが、未分類ゾーンではなく Delete 象限に誤って配置される不具合を修正（未リリースの #19 機能の不具合）。実機 Obsidian の欠損値（`NullValue`）判定を文字列表現依存から型同一性（`instanceof`）へ是正し、実機 E2E で未分類化を確認した。(#33)
- 軸に非 boolean のプロパティ（数値・文字列の `note.*`）を指定したノートが 4 象限に分類されてドラッグ可能になり、ドロップで両軸が `true`/`false` に上書きされて元の値を破壊しうる不具合を修正（未リリースの #19 機能の不具合）。v1 は boolean 軸限定のため、値が boolean のプロパティだけを分類し、非 boolean（および absent）のノートは未分類ゾーンへ退避するようにした。(#34)
- Base 自身の `.base` ファイルや `.canvas`・画像などの非 Markdown が「未分類」カードとして現れうる問題を修正（未リリースの #19 機能の挙動改善）。配置対象を Markdown ノート（`file.extension === "md"`）のみに限定し、非 Markdown は象限にも未分類にも出さないようにした。Markdown ノートが 1 件も無い Base は空状態を表示する。
- 軸に数値・文字列など boolean 以外の値を持つノートを未分類ゾーンから象限へドラッグすると、両軸が `true`/`false` で上書きされ元の値が壊れる問題を防いだ（未リリースの #19/#34 機能の穴を封鎖）。そうしたノートは未分類ゾーンで淡色＋鍵アイコン付きの**ドラッグ不可カード**として表示し、誤ドロップによるデータ破壊を防ぐ（クリックで開いて値を直せる）。軸プロパティが未設定（欠損）のノートは従来どおりドラッグして分類できる。
- 表示まわりの細かな改善（いずれも未リリース機能の詰め）: Bases が無効な環境での登録失敗メッセージも表示言語（英/日）に追従するようにした。象限の件数バッジをスクリーンリーダーが読み上げられるようにした。英語表示時の件数を単複で正しく出す（`1 item` / `5 items`）。
- さらに未リリース機能の詰め: カードを移動した直後に「元に戻す」トーストで取り消すと、カードが移動先の象限に貼り付いて見える場合があったのを修正（ファイル自体は正しく戻っていた）。緊急度・重要度の両軸に同じプロパティを割り当てるとドラッグで値が壊れうるため、その設定では移動を通知して弾くようにした。未分類ゾーンを非表示にしていて表示できるカードが全て未分類のとき、画面が無言で空になる代わりに件数ヒントを出すようにした。設定の言語「Auto」を日本語 UI では「自動」と表示。ビューを狭いペイン（サイドバー等）に置いたときも 2×2 が 1 列に畳まれるよう、ペイン幅に追従するようにした。「元に戻す」トーストのボタンをキーボードで操作した後、フォーカスが失われずマトリクスへ戻るようにした。
- カードをドラッグ中、カーソルに追従する浮遊カード（オーバーレイ）が実際のカーソル位置から一定量ずれて表示される問題を修正（未リリースの #20 機能の不具合）。ビューを中央ペイン・サイドパネルのいずれに置いても、掴んだ位置にオーバーレイが正しく追従するようにした。(#43)
- ビューをポップアウトウィンドウ（別ウィンドウ）に出すとカードを掴めずドラッグを開始できない問題を修正（未リリースの #20 機能の不具合）。別ウィンドウでもカードを掴んで象限を移動できるようにした（マウス・キーボード操作とも）。実機 Obsidian で修正前後を確認（別ウィンドウでの掴み: 不可→可、メインウィンドウは変化なし）。(#44)
- 収束レビュー（多観点）対応（いずれも未リリース機能の詰め）: 「元に戻す」がノートをパスで再解決する際、移動後にそのパスが別ノートで再利用されていたり軸値が外部で書き換えられていた場合に無関係な frontmatter を上書き/削除しうる経路を塞いだ。記録したノートが削除・リネームされた時点で取り消し記録を破棄し（パス再利用を根本から遮断）、加えて復元前に「記録時に書き込んだ値と現状が一致する場合のみ戻す」同一性照合を行う 2 段構え（取り消しは唯一の削除経路のため二重に安全側へ）。ドラッグ不可（ロック）カードのタイトル文字が薄くなりすぎてコントラストが低下していたのを、不透明度ではなくテーマの弱色トークンで表現するよう変更し可読性（WCAG AA）を確保。「元に戻す」トーストの自動消滅（8 秒）を、フォーカス/ポインタがトースト内にある間は一時停止し、両方が離れたら再開するようにして、キーボード/支援技術の利用者がボタンへ到達する前に消えないようにした（WCAG 2.2.1）。開発者向けコンソールログの言語を統一。緊急度・重要度の両軸に同じプロパティを割り当てた設定ミスのとき、カードが象限に並んで掴めるのにドロップが必ず失敗し通知が出続ける問題を、当該ビューの全カードをドラッグ不可（ロック）にして解消。ドラッグ不可（ロック）カードを Space キーで操作しても無反応でペインがスクロールするだけだったのを、Enter と同じく「開く」に割り当て既定のスクロールを抑止するよう是正（ボタンの標準操作に統一）。あわせて取り消し記録の無効化・値の同一性照合・タイマー再開判定・同一象限への無駄打ち判定・キーボードドラッグ中の Enter 抑止・両軸同一キーのロック・ロックカードの Space 操作に回帰テストを追加。
